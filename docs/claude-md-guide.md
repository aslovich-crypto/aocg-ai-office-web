Скопируйте блок ниже в файл `CLAUDE.md`, удалите этот вводный абзац и заполните `<плейсхолдеры>` под свой проект. Держите файл коротким: только то, что агент должен знать всегда. Разовые задачи — в промпт, не сюда.

## Как поддерживать `CLAUDE.md`

- **Коротким.** Раздутый контекст вредит так же, как его отсутствие. Если файл разросся — вынести детали в отдельные доки и оставить ссылки.
- **Актуальным.** Сменили стек, добавили конвенцию, поняли новый запрет — сразу в `CLAUDE.md`. Это часть фичи, не отдельная задача.
- **Версионируемым.** Файл в git, проходит code review вместе с кодом.
- **Разделённым.** Постоянное — здесь. Разовое — в промпт. Методология и история — в отдельные документы.

## Почему это работает

Агент перестаёт нарушать стиль и конвенции, потому что видит их всегда. Промпты становятся короче — не нужно повторять бренд и запреты. Контекст не теряется между сессиями. Это самый дешёвый рычаг повышения качества вывода из всех доступных.
