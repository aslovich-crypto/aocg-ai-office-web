# Постановка UI-фичи

При запросе UI-правки — раскрыть задачу в шесть блоков. Бренд-констрейнты не повторять (они в CLAUDE.md), только сослаться.

## Шесть блоков
1. **Контекст** — какой экран/компонент, зона работы.
2. **Задача** — по пунктам, конкретно, с текущим и целевым состоянием каждого.
3. **Целевое состояние** — как наблюдаемо проверить, что готово.
4. **НЕ трогать** — логика, state, API, другие экраны, общая шапка.
5. **Констрейнты** — «следуй CLAUDE.md» + разовые ограничения этой задачи.
6. **Режим** — «сначала план, не реализуй до одобрения; затем diff; коммит отдельной командой».

## Каркас
```
Контекст: <экран / компонент>. Зона работы — только это.

Задача:
  1. <что меняем>
     - текущее: <...>
     - целевое: <...>

Целевое состояние:
  <наблюдаемая проверка>

НЕ трогать:
  - логику, state, запросы к API
  - другие экраны и общую шапку

Констрейнты:
  - следуй CLAUDE.md (дизайн-система, UX-принципы)

Режим:
  сначала план, код не пиши до одобрения; затем diff, не коммить.
```

## Правило
Не принимать к работе размытую задачу («сделай красиво», «как у другого продукта»). Сначала развернуть в шесть блоков, при нехватке данных — уточнить, и только потом план.
